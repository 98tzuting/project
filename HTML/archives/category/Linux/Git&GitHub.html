Git&GitHub

: Posted on 2022-09-15

Git常用來保存與備份，以免檔案流失，通常會用腳本去自動執行

切記數據都要先下載再開始作業以免檔案重疊被覆蓋

首先還是需要下載，分為下載ZSH、下載oh-my-zsh、更改主題、設置簡單alias、下載插件plugins
這裡我使用為Ubuntu 來做示範，WSL遠端Ubuntu內建已有Git
若在本機則需要下載Git至本機+設定

前置作業設定識別資料 :
設定帳戶，讓 Git 知道電腦做的修改要連結到哪位使用者，建議跟Emailㄧ致的帳號 
 git config --global user.name ""
設定Git-Email 
 git config --global user.email ""
初始化git 
 git init
初始提交基本東西 
 git commit -m "Initial Commit"
設定推送地點 
 git remote add origin ssh://git@.......
設定推送分支 
 git push -u origin master
再次查看狀態 
 git status
使用git發佈順序 :
查看狀態 
 git status
新增要推送的檔案 
 git add . 與  git add --all 相同
推送版本訊息 
 git commit -m "版本訊息(時間或是人物名稱master)"
推送至Git 
 git push
再次查看狀態 
 git status
忽略的文件 gitignore :
忽略的文件 gitignore bin匹配任何名為“bin”的文件或目錄。
bin/匹配任何名為“bin”的目錄，這實際上意味著它的所有內容，因為 Git 不會單獨跟踪目錄。
bin/*直接匹配 any 中的所有文件和目錄bin/。這可以防止 Git 自動在其子目錄中查找任何文件，但如果bin/foo創建了子目錄，則此規則將不匹配foo的內容。
bin/**匹配任何bin/目錄及其所有子目錄中的所有文件和目錄。
